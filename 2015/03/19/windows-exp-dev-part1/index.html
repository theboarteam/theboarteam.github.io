CPU寄存器

CPU寄存器就像是一个小的存储空间，用来快速的存取数据。在X86（32位）架构中，这里一般有8个一般目的寄存器：EAX、EBX、ECX、EDX、EDI、ESI、EBP和ESP。从技术角度说，他们能够用来存储任何数据，尽管最初他们是被设计用于执行特定任务的，但是在很多情况下，仍然如现在这样来使用。

这里有一些更加详细的关于他们的介绍…

EAX —— 累加寄存器

它叫做累加寄存器是因为最开始的时候就是用于做加减计算的（指令类似 ADD 和 SUB）。当其他寄存器可以被用于计算时，EAX被分配到给予更有效率的一字节OPCODE操作中。当缓冲区非常有限制的时候，这种特性是非常重要的，尤其在以后写Exploit当中更能体现。除此之外EAX还用于存储函数执行后的返回值。
这个一般目的寄存器能被部分和整个引用，以如下方式：EAX完整的是指的32位寄存器，而AX则代表了小端的16位寄存器，它同时能被分拆位AH（高8位）和AL（低8位）。

这种整体/部分的特性 32-、16-、8-位引用同样被用于其他下面三种寄存器（EBX、ECX、EDX）

EBC —— 基址寄存器

在32位架构种，EBX并没有明确的特别用途，所以只要把它当做所有有效存储的汇总就可以了。像EAX奕扬，它也能以不同方式引用（整个EBX，部分BX、BH、BL）

ECX —— 计数寄存器

就像它的名字所示，计数寄存器被频繁的用于循环结构种，当然它也能用来存储任何数据，也可以以不同方式来引用（跟前面所说相同）。

EDX —— 数据寄存器

EDX像是EAX的搭档一样，它经常用于精确的数学操作，像乘除法中存在溢出时，高位会被放在EDX，低位会被放在EAX。它同样用于存储函数的变量，同时像EAX有多种引用方式。

ESI —— 来源寄存器

作为EDI的副本，ESI通常用于存储读取地址的指针。举个例子，如果一个函数被设计用于读取字符串，ESI将会把整个字符串起始地址存入到指针种，存的是指针的地址。

EDI —— 目标寄存器

尽管它能用于一般的数据存储，但是EDI最初是被设计用于存函数地址的指针的，像一些写地址的字符串操作。

EBP —— 基址指针寄存器

EBP是用来保存堆栈的基/底的现场。它经常被用来引用位于栈中的变量以抵消当前的EBP的值，如果参数是通过寄存器来传值的话，你可以选择使用EBP作为一般用途。

ESP —— 栈指针寄存器

ESP用于保存栈顶现场。随着栈内项目相应地移动，ESP 也会随之递增/递减。所有的通用寄存器中，ESP 很少被用于其他用途。

EIP —— 指令指针寄存器

不是一个通用寄存器。但是在这块种， EIP 指向的内存中CPU即将执行的下一条指令的地址。你会看到在接下来的教程中，控制了 EIP 的值，您可以控制执行流的应用程序 （以执行你所选择的代码）。

段寄存器（Segment Register）和EF标志位寄存器（EFFLAGS Register）


还有有两个额外的寄存器，你会看到他们在寄存器窗口栏种，段寄存器和 EF标志位寄存器。我将不会讨论到太多关于他们的细节，但是要注意到 EF标志位寄存器组成一系列标志的意义，是用于表示计算结果种的布尔值变量的，并可以同时用于确定当特有条件成立后所发生的跳转（以后会讨论更多这一方面）。

CPU指令集

可能你已经发现了，绝大部分现代的程序都是使用高级语言编写的（C、C++等）。当程序被编译完成，这些高级语言代码就被翻译成汇编语言，人们有对应的OPCODE来帮助我们理解进一步转换成的机器代码。在调试器种，你能够看到每一句汇编指令（和对应的OPCODE），从CPU指令面板中看到CPU正执行到哪一句。注：关于这个系列，我使用的是Intel x86汇编语言（ http://en.wikipedia.org/wiki/X86_assembly_language ）。
你能使用F7步过每一个程序的执行流程，然后看看每一条CPU指令执行后的结果。让我们瞅一眼WMP的第一句指令吧。这个程序在开始时候会暂停下来，然后我们按几次F7来让它执行起来，知道我们到达第二个 MOV DWORD PTR SS: 指令处（截图中高亮部分）。MOV指令从一个地址拷贝了一些数据到另一个地址。

这条指令将移动EBX中的内容到EBP - 18 指向的内存地址中去（在Interl x86语法中它是 MOV [dst] [src]）。我们注意到EBP（栈基指针）此时正指向0007FFC0。使用你的Windows或者Mac计算器（科学/程序员模式），计算下0007FFC0 - 0x18是多少。结果将会是0x7FFA*，这意味着EBP的值将会被替换成0007FFA8这个地址。实际上，你不需要计算这些东西，因为在CPU指令面板中的子窗口，已经会告诉你计算后的值了，包括它之后的地址的值。你可以在子窗口中使用右键单击“栈”的那一行，选择“在Dump种跟随地址”来验证这个内存地址的值。

现在让我们再按下F7，注意内存地址0007FFA8是怎么变化到00000000的，当EBX的值被传到这儿之后。

这里只是一个简单快速的例子来教会你如何跟随CPU指令，这里有一些经常会用到的汇编指令和语法，你应该掌握他们：

  • ADD/SUB op1,op2 —— 两个操作数加减操作，会将结果存储在第一个操作数中。这些可以是寄存器，内存地址，或者是个常数。举例来说，ADD EAX,10 意味着加了10到EAX中，并最终将结果存储在EAX种。
  • XOR EAX.EAX —— 就是清零寄存器用的。最简单的方式了吧～
  • INC/DEC op1 —— 增加或者减少某个操作数的值，一次是1哦～
  • CMP op1,op2 —— 比较两个操作数的值（寄存器/内存地址/常数），然后设置某个标志位寄存器的值。
  • 跳转（JMP）和条件跳转（JE、JZ等等）—— 就像这个指令的名字一样，它能够任意的跳转到某个地址。而条件跳转则是发生在某些特别的情况下的（使用标志位寄存器来作为判断）。比如，它可能会比较两个寄存器的值，然后跳到某个地址，如果这俩值相等（JE指令，当zf标志位的值为0时，则跳转）。
  • 当你看见一个值类似 ADD DWORD PTR [X] 或者 MOV EAX,[EBX]，它代表了存在某个内存地址的值X。换句话说，EBX中的内容就是[EBX]所存储的地址的值。（这里翻译不太好，就是指针的概念啦～）
  • 大小关键字： BYTE = 1byte，WORD = 2bytes，DWORD = 4bytes

PEB和TEB

当你运行一个程序的时候，一个可执行文件的实例，我们叫做进程的东西就起来了。每个进程提供了一些必要的资源来启动这个程序。每个Windows进程都有一个 可执行进程结构（EPROCESS），它包含了进程的一些属性，重定向指针结构等。大部分这些EPROCESS结构体是放在内核空间中的，进程环境块（PEB）是放在用户态内存中的。PEB包换了各种用户态下运行起来一个程序的相关参数。你可以使用WinDbg来很容易的查看PEB结构的内容，使用!peb命令就可以了。

正如你所见，PEB包含了如下信息，包括PE文件的映像基址，堆分配的地址，和所载入的DLL，环境变量等等。从上面Windbg的截图来看一下映像基址（Image Base Address）。注意到01000000这个地址。现在回顾之前看到的Win32内存映射关系图，请注意他们是如何对应在”程序映像”块中相同的第一个地址的。同样你能按照这个关系找到堆分配地址和DLL地址。

一个程序或者进程，可以有一个或者多个线程，他们会一直跑着直到操作系统分配完给处理器处理的时间。每个进程都至少有一个主线程，但是如果需要的话还可以创建其他线程。所有这些线程共享同一个虚拟地址空间和系统分配给父进程的处理器时间。每个线程同样有其自己的资源信息，包括异常处理句柄，优先级、本地存储等等。就像每个进程都有PEB结构一样，每个线程也同样有TEB结构。TEB存储了映像载入器和各种DLL的上下文信息，以及异常处理程序列表 （我们将在后面的帖子中详细介绍） 的地址。TEB 像PEB那样驻留在进程的地址空间，只要用户模式组件需要写访问权限的时候。

堆

堆是由程序员自己直接分配的内存空间（使用malloc()函数），通常用来存储全局变量。它不同于栈，堆内存分配必须由程序管理。换而言之，该内存分配将保持，直到它被释放由程序或程序本身即告终止。你可以认为堆是一个共享的内存池，而堆栈中，我们将讨论下一步，更是组织和条块分割。这里不会对堆做过多讨论，可能将会在以后的章节中深入。

栈

不同于堆，在全局变量的内存分配是相对任意性和持久性，该堆栈用于本地分配短期存储（函数/方法）在一个有序的方式变量和内存然后在给定的函数的终止释放。记得有一个给定的进程可以有多个线程。每个线程/功能分配自己的堆栈帧。该堆栈帧被固定后的创作和栈帧是在函数的结论删除尺寸。

栈帧和函数

当一个程序的函数执行时，单个栈帧被创建用于存储本地变量。每一个函数都会有其自己的栈帧，这是放在当前栈，使栈从低地址向上生长。
每次一个栈帧被创建，一些列的操作执行来存储函数的参数和返回地址（简称保存现场，这样程序就知道当函数执行结束的时候返回到哪儿继续运行），保存当前栈的基址指针，然后恢复本地变量所要的空间（SUB操作）【注：我有意的忽略了异常处理，这将会后面才讨论】

你能看到一些值被倒过来写1C30300.让我们抓住这次机会来快速看一眼小端字节序。”端”指的是内存中各字节排列的顺序。Intel X86 架构是以小端序来排列的，其存储一个值，在最小的内存地址 （这就是为什么的地址以相反的顺序存储） 的最低有效字节。举一个例子，指向前面的16进制dump截图 —— 是最小的顶部 （00332FD4） 地址和地址在底部 （00333034） 是最大。因此中左上角, 的字节 （目前占用 1 C） 占用的最小的地址位置和地址获得更大的当您移动左到右，从上到下。当你看着地址 0033301C 时，最低有效字节是一路向右 (1c) 字节。将其转换为小端字节序符号你重新排序它，一次一个字节从右到左。这里是一个虚拟演示：

一旦foo()函数被调用，首先会发生的是，当前基地址指针（EBP)被保存到栈，通过PUSH EBP指令。所以一旦被调用的函数终止，main()函数就能回到原来调用的地方继续运行。
接下来，EBP被设定为与ESP相同(通过MOV EBP,ESP指令)，这使得栈底部和顶部的栈帧相同。从这里，EBP将会保持位一个常量(foo()函数的生命周期），并且ESP 将升到较低的地址，把数据添加到该函数的堆栈帧。这里是之前和之后显示 EBP 现在等于 ESP 寄存器的视图。

函数main() 可以通过移动堆栈指针在堆栈 （通过增加其价值 4） 和清理堆栈的 argv [1] 来执行他们自己的清理。它将清除该登记册，它用来存储 argv [1] （EAX） 通过 XOR、 还原已保存EBP，然后返回到保存的返回地址。
